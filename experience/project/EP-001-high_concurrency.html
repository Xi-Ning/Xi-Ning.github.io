# 1 某高并发互联网项目

某高并发互联网项目
创建时间： 2021.04.19
竢实扬华 · 自强不息

# 1.1 项目特点

项目为互联网项目，在业务高峰期，瞬时并发高，持续压力大，且不容许在发生任何安全事件。

# 1.2 三级缓存设计

为应对访问压力，保护数据库，系统在缓存方面设计为三层：

  • 第一级：购买CDN服务，将JS、CSS与图片等静态资源、下载服务及门户网站服务的查询请求挂载至CND

  • 第二级：使用Nginx服务器，缓存门户网站服务的查询请求，与业务服务的部分查询请求

  • 第三级：使用Redis服务器，缓存门户网站服务的查询请求，与业务服务的部分查询请求，以及其他系统基础配置信息

在程序层面，封装出通用的Redis缓存工具类，并为所缓存的数据增加缓存信息，如下图所示：

其中：

  • cacheTime代表着数据进入Redis服务器的时间，也就是请求成功访问了数据库的时间

  • achieveTime代表着从Redis中获取数据的时间，也代表这请求抵达了应用服务器但未抵达数据库，如果这个时间从前端观察不发生变化，则说明请求没有抵达应用服务器，而是由CDN或Nginx所返回

# 1.3 多维度的监控手段

项目采用微服务架构体系，由40余台虚拟机支撑。因此必须有足够的监控手段来时时掌握虚拟机运行状况，及时发现问题、解决问题。

# 1.3.1 nmon

nmon是由IBM提供，免费监控AIX系统与Linux系统资源的工具。既可以在一个屏幕上显示所有重要的性能优化信息，并动态地对其进行更新，还可以将相同的数据捕获到一个文本文件，便于以后对报告进行分析和绘制图形。

对于我们来说，以下指标更为关心：

  • CPU使用率

  • 内存使用情况

  • 磁盘I/O速度、传输和读/写比率

  • 文件系统中的可用空间

  • 网络I/O速度、传输和读/写比率

# 1.3.2 curl

curl命令常被用来抓取网页和监控Web服务器状态，我们可以利用该命令，编写出快速检测所有应用运行状态的shell脚本，部分脚本内容如下：

STATUS_200='\033[32m正常\033[0m'
STATUS_000='\033[31m宕机\033[0m'

WEB_GK_AUTH_NODES=(xx.xx.xx.xx1:9998 xx.xx.xx.xx2:9998 xx.xx.xx.xx3:9998 xx.xx.xx.xx4:9998)

echo '应用单点检测'

for node in ${WEB_GK_AUTH_NODES[@]}
do
HTTP_CODE=$(curl 'http://'$node'/xx/xx/xx/token' -o /dev/null -s -w %{http_code})
if [ $HTTP_CODE == '200' ]; then
echo -e 'xx-xx-auth  '$node'  ------------------------------------>  ' $STATUS_200
elif [ $HTTP_CODE == '000' ]; then
echo -e 'xx-xx-auth  '$node'  ------------------------------------>  ' $STATUS_000
else
echo -e 'xx-xx-auth  '$node'  ------------------------------------>  \033[33m' $HTTP_CODE'\033[0m'
fi
done

# 1.3.4 zabbix

zabbix是一个基于web界面的提供分布式系统监视以及网络监视功能的企业级的开源解决方案。

我们利用zabbix可以全面的对系统进行监控，不仅仅限于每一台虚拟机的CPU、硬盘、网络、内存，还可以通过编写脚本，监控Nginx的活跃连接数，Redis的活跃连接数、内部key的数量、内部内存使用情况等等。

# 1.4 Linux参数调优

Linux基础参数的设置，在很大程度上会影响高并发场景下的应用性能。

# 1.4.1 系统级文件/etc/security/limits.conf

该文件中增加如下参数，用以设置系统可以打开的最大进程数与文件数：

*    soft    nproc     65535
*    hard    nproc     65535
*    soft    nofile    65535
*    hard    nofile    65535

# 1.4.3 系统级文件/etc/sysctl.conf

该文件中增加如下TCP性能调优参数：

#开启TCP连接中time_wait sockets的快速回收
net.ipv4.tcp_tw_reuse = 1
#开启TCP连接复用功能，允许将time_wait sockets重新用于新的TCP连接
net.ipv4.tcp_tw_recycle = 1
#如果套接字由本端要求关闭，这个参数决定了它保持在FIN-WAIT-2状态的时间
net.ipv4.tcp_fin_timeout = 30
