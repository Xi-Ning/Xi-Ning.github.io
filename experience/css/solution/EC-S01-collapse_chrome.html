# 1 border-collapse:collapse在Chrome下的bug

collapse在Chrome下的bug
创建时间： 2022.01.17
竢实扬华 · 自强不息

Chrome浏览器下，表格边框出现了粗细不一的情况：

复现此问题的两个必要条件：

  • 表格上做了合并单元格（colspan/rowspan）

  • 通过border-collapse:collapse属性设置了表格边框合并：

table {border-collapse: collapse;}
th, td {border: 1px solid #868686;}
1
2

Chrome官网上，在此错误报告下的回复：

It's a known (old) issue in our table code. Collapsing borders are determined based on adjacent cells and our code doesn't deal correctly with spanning cells (we only consider the cell adjoining the first row / column in a row / column span). On top of that, our border granularity is determined by the cell's span.

To fix this bug, we would need to overhaul our collapsing border code, which is a big undertaking.

既然Chrome没有修复此问题，我们只能通过其他方式间接解决边框问题。

方法一：

核心思路：

  • 边框分离模式（border-collapse: separate）下，设置边框间距为0

  • table增加上边框与左边框

  • th/td增加下边框与右边框

table {border-collapse: separate; border-spacing: 0; border-top: 1px solid #868686; border-left: 1px solid #868686;}
th, td {border-right: 1px solid #868686; border-bottom: 1px solid #868686;}
1
2

方法二：

核心思路：

  • 边框分离模式（border-collapse: separate）下，设置边框间距为1px

  • 不设置实际的边框，table与th/td分别设置背景色，通过背景色的差异形成边框的效果

table {background-color: #868686; border-collapse: separate; border-spacing: 1px;}
th, td {background-color: #ffffff;}
