# 1 exp 导出空表

exp 导出空表
创建时间： 2021.03.31
竢实扬华 · 自强不息

# 1.1 问题分析与解决方法

问题描述：Oracle 11g及以后的版本，在导出数据库时，会自动过滤掉空表。

原因：Oracle出于节省空间的考虑，默认对空表不分配segment。

解决方法：手工为空表分配extent。

alter table table_name allocate extent;
1

关于segment、extent和block

segment（段）、extent（区）、block（块）都是Oracle存储的物理概念名称。

  • block是Oracle存储的最基本单位，由db_block_size指定，通常为8KB，由连续的sector（磁盘的最小存储单位，512byte）组成

  • 连续的数据块block组成一个分区extent，便于空间管理，包括空间的分配和释放

  • segment段的由extent区组成，但不要求连续

# 1.2 批量处理空表

通过以下语句，可以查询到当前用户下的所有空表：

select table_name from user_tables where num_rows = 0 or num_rows is null;
1

进而可以为所有空表批量生成分配extent的语句：

select 'alter table ' || table_name || ' allocate extent;' from user_tables where num_rows = 0 or num_rows is null;
1

将结果集导出并执行即可。

# 1.3 脚本工具的编写与使用

（1）创建一个本地脚本文件init.sql，内容如下：

set heading off;
set echo off;
set feedback off;
set termout on;
spool d:\allocate.sql;
select 'alter table ' || table_name || ' allocate extent;' from user_tables where num_rows = 0 or num_rows is null;
spool off;
