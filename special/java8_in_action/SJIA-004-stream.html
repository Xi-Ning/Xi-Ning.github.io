# 4.1 流是什么

# 4.1.1 Java 7与Java 8的对比

流是Java API的新成员，它允许以声明性方式处理数据集合（通过查询语句来表达，而不是临时编写一个实现）；流还可以透明地并行处理，无需写任何多线程代码。

Java 7典型的指令型编程方式：

List<Dish> lowCaloricDishes = new ArrayList<>();
for(Dish d : menu) {
    if(d.getCalories() < 400) {
        lowCaloricDishes.add(d);
    }
}
Collections.sort(lowCaloricDishes, new Comparator<Dish>() {
    public int compare(Dish d1, Dish d2) {
        return Integer.compare(d1.getCalories(), d2.getCalories());
    }
});
List<String> lowCaloricDishesName = new ArrayList<>();
for(Dish d : lowCaloricDishes) {
    lowCaloricDishesName.add(d.getName());
}

用Java 8的流来重构这段代码：

import static java.util.Comparator.comparing;
import static java.util.stream.Collectors.toList;
List<String> lowCaloricDishesName = 
                    menu.stream()
                        .filter(d -> d.getCalories() < 400)
                        .sorted(comparing(Dish::getCalories))
                        .map(Dish::getName)
                        .collect(toList());

Java 8这段代码是以声明性方式写的，这种方式加上行为参数化，可以轻松的应对变化的需求。另外，这段代码将几个基础操作链接起来，来表达复杂的数据处理流水线，同时保持代码清晰可读。

TIP

因为filter、sorted、map和collect等操作是与具体线程模型无关的高层次构件，所以它们的内部实现可以是单线程的，也可能透明地充分利用多核架构。 在实践中，这意味着你用不着为了让某些数据处理任务并行而去操心线程和锁了，Stream API都替你做好了。

总结，Java 8中的Stream API可以写出这样的代码：

  • 声明性：更简洁，更易读

  • 可复合：更灵活

  • 可并行：性能更好

# 4.1.2 流的定义

流是从支持数据处理操作的源生成的元素序列。

  • 元素序列：像集合一样，流也提供了一个接口，可以访问特定元素类型的一组有序值。

  • 源：流会使用一个提供数据的源，如集合、数组或输入/输出资源。（从有序集合生成流时会保留原有的顺序）

  • 数据处理操作：流的数据处理功能支持类似于数据库的操作，以及函数式编程语言中的常用操作，可以顺序执行，也可以并行执行。

# 4.1.3 流的重要特点

  • 流水线：很多操作本身会返回一个流，这样多个操作就可以链接起来，形成一个大的流水线。这样一些优化成为可能，如延迟和短路。

  • 内部迭代：与使用迭代器显式迭代的集合不同，流的迭代操作是在背后进行的。

# 4.2 流与集合

# 4.2.1 关注点不同

集合与流都提供了访问元素类型有序值的数据接口。

集合是数据结构，主要目的是以特定的时间/空间复杂度存储和访问元素，关注的是数据；而流的目的在于表达计算。

# 4.2.2 计算时机差异

集合是一个内存中的数据结构，它包含数据数据结构中目前所有的值——集合中的每个元素都得先算出来才能添加到集合中。（集合可以添加或删除元素，但不论何时，集合中的元素都是放在内存中的，元素必须先计算出来才能被添加到集合中。）

流则是概念上固定的数据结构（不能添加或删除元素），元素是按需计算的，流像是一个延迟创建的集合，只有在消费者要求的时候才会计算值。

# 4.2.3 只能遍历一次

流和集合的迭代器类似，只能遍历一次，遍历一次后流已经被消费掉了，可以从原始数据源那再获得一个新的流来重新遍历。

# 4.2.4 遍历方式不同

使用Collection接口需要用户去做迭代，这称为外部迭代，需要显示地取出每个项目再加以处理。

Streams库使用内部迭代——它帮你把迭代做了，还把得到的流值存在了某个地方，你只要给出一个函数说要干什么就可以了。内部迭代可以自动选择一种适合硬件的数据表示和并行实现。

// 外部迭代：迭代器
List<String> names = new ArrayList<>();
Iterator<String> iterator = menu.iterator();
while(iterator.hasNext()) {
    Dish d = iterator.next();
    names.add(d.getName());
}

// 外部迭代：for-each语法糖
List<String> names = new ArrayList<>();
for(Dish d : menu) {
    names.add(d.getName());
}

// 内部迭代：流
List<String> names = menu.stream()
                         .map(Dish::getName)
                         .collect(toList());

# 4.3.1 中间操作

可以连接起来的流操作称为中间操作。

中间操作可以返回另一个流，这让多个操作可以连接起来形成一个查询。

除非流水线上触发一个终端操作，否则中间操作不会执行任何处理，这是因为中间操作一般都可以合并起来，在终端操作时一次性全部处理。

将上面的代码改造一下进行输出，来观察求值的顺序：

List<String> names = menu.stream()
                         .filter(d -> {
                                           System.out.println("filtering " + d.getName());
                                           return d.getCalories() > 300;
                                      })
                         .map(d -> {
                                        System.out.println("mapping " + d.getName());
                                        return d.getName();
                                   })
                         .limit(3)
                         .collect(toList());
System.out.println(names);

# 4.3.2 终端操作

关闭流的操作称为终端操作。

终端操作会从流的流水线生成结果，其结果是任何不是流的值，甚至void。

# 4.3.3 使用流

流的使用一般包含三件事：

  • 一个数据源（如集合）来执行一个查询

  • 一个中间操作链，形成一条流的流水线

  • 一个终端操作，执行流水线，并能生成结果

流的流水线背后的理念类似于构建器模式。在构建器模式中，有一个调用链用来设置一套配置，接着调用built方法。

Stream API提供的部分操作：

操作 类型 返回类型 操作参数 函数描述符
filter 中间 Stream<T> Predicate<T> T -> boolean
map 中间 Stream<T> Function<T, R> T -> R
limit 中间 Stream<T>
sorted 中间 Stream<T> Comparator<T> (T, T) -> int
distinct 中间 Stream<T>
操作 类型 目的
forEach 终端 消费流中的每个元素并对其应用Lambda。这一操作返回void。
count 终端 返回流中元素的个数。这一操作返回long。
collect 终端 把流归为一个集合，比如List、Map甚至是Integer。
