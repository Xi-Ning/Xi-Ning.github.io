# 2.1.1 套路化的集合处理

筛选（filter）指定颜色的苹果：

public static List<Apple> filterApplesByColor(List<Apple> inventory, String color) {
    List<Apple> result = new ArrayList<>();
    for(Apple apple : inventory) {
        if(apple.getColor().equals(color)) {
            result.add(apple);
        }
    }
    return result;
}

public static List<Apple> filterApplesByWeight(List<Apple> inventory, int weight) {

            result.add(apple);

弊端分析

  • 软件工程中复制粘贴会带来的风险：给一个做了更新和修正，却忘了另一个。

  • 违反了软件工程原则：DRY（Don't Repeat Yourself，不要重复自己）。

如果将所有的属性都结合起来形成一个筛选方法，那么方法会十分的笨拙，客户端的代码也会十分糟糕。比起将颜色、重量，甚至逻辑标记位等属性传递给筛选方法，我们更应该去考虑传递苹果的筛选标准。

# 2.1.2 行为参数化

定义一个接口对筛选标准进行建模，也就是根据苹果的某些属性来返回一个boolean值，也就是谓词：

public interface ApplePredicate {
    boolean test(Apple apple);
}
1
2
3

predicate（谓词）

谓词在数学上常常用来代表一个类似函数的东西，它接受一个参数值，并返回true或false。

用ApplePredicate的多个实现代表不同的选择标准，通过增加ApplePredicate的实现，就可以创建出不同的筛选策略：

public class AppleHeavyWeightPredicate implements ApplePredicate {
    public boolean test(Apple apple) {
        return apple.getWeight() > 150;
    }
}

public class AppleGreenColorPredicate implements ApplePredicate {
    public boolean test(Apple apple) {
        return "green".equals(apple.getColor());
    }
}

策略设计模式

ApplePredicate可以称为算法族，而AppleHeavyWeightPredicate和AppleGreenColorPredicate就是该族算法下不同的策略。

让filter方法接收ApplePredicate对象，这样就将集合迭代的逻辑与要应用到集合中每一个元素的行为区分开了：

public static List<Apple> filterApples(List<Apple> inventory, ApplePredicate p) {
    List<Apple> result = new ArrayList<>();
    for(Apple apple : inventory) {
        if(p.test(apple)) {
            result.add(apple);
        }
    }
    return result;
}

行为参数化

让方法接受多种行为（或策略）作为参数，并在内部使用，来完成不同的行为。

现在，筛选的客户端代码会是这样的：

List<Apple> heavyApples = filterApples(inventory, new AppleHeavyWeightPredicate());

# 2.1.3 使用匿名类

行为参数化后，如果想要将新的行为传递给filter方法时，不得不声明多个ApplePredicate的实现类，然后实例化多个可能只会使用一次的实例对象。我们可以使用匿名类进一步简化代码：

List<Apple> greenApples = filterApples(inventory, new ApplePredicate() {
    public boolean test(Apple apple) {
        return "green".equals(apple.getColor());
    }
});

匿名类

可以同时声明和实例化一个类。

匿名类还是不够友好，它往往很笨重（存在很多模板代码），占用了很多空间。并且代码的易读性没有很好，不那么一目了然。

# 2.1.4 使用Lambda表达式

在Java 8中，可以使用Lambda表达式，使代码更为干净、清晰：

filterApples(inventory, (Apple apple) -> "green".equals(apple.getColor()));
filterApples(inventory, (Apple apple) -> apple.getWeight() > 150);
filterApples(inventory, (Apple apple) -> apple.getWeight() < 80 && "green".equals(apple.getColor()));

# 2.1.5 集合类型抽象化

public interface Predicate<T> {
    boolean test(T t);
}

public static <T> List<T> filter(List<T> list, Predicate p) {
    List<T> result = new ArrayList<>();
    for(T e : list) {
        if(p.test(e)) {
            result.add(e);
        }
    }
    return result;
}
